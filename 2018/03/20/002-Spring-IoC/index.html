在所有的 Java 程序中，总存在 A 对象调用 B 对象， B 对象调用 C 对象，这种情形被 Spring 称为依赖，即 A 对象依赖 B 对象，B 对象依赖 C 对象。对于整个 Java 项目而言，它是由一些相互调用的对象所构成， Spring 把这种相互调用的关系称为依赖关系。

Spring 提供的核心功能有两个：

  1. Spring容器作为一个超级大工厂，负责创建、管理所有的Bean（Java对象）；
  2. Spring容器管理容器中Bean之间的依赖关系，它使用一种称为“依赖注入”的方式来管理 Bean 之间的依赖关系。

使用依赖注入，程序中的各个组件不需要以硬编码的方式耦合在一起，同时不再需要工厂模式，我们只需使用配置文件就可将各个组件组织在一起。使用依赖注入，不仅可以为Bean注入普通的属性值，还可以注入其它 Bean 的引用。
